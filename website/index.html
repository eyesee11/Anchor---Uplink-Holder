How It Works

01
👁

Watch

Run anchor add-folder "C:\Videos". Anchor registers every existing file and hooks into the Windows OS file event system (ReadDirectoryChangesW) for new files. Zero CPU polling between events.

02
#

Hash

Every file is fingerprinted with SHA-256 using a native C binary — 3–5× faster than Java for large files. The hash and file size are stored in a local SQLite database, so Anchor never re-uploads an unchanged file.

03
☁

Upload

Files are uploaded to Google Drive in 8 MB chunks. After each chunk, the resume URI and byte offset are saved to disk. If your connection drops, Anchor resumes from the last confirmed chunk on the next run.

Why Anchor

Google Drive Desktop / Browser

  • Restarts upload from byte 0 on any network failure
  • No visibility into chunk-level upload progress
  • Background sync client — heavy on resources
  • No CLI interface for scripting or automation
  • Can't target a specific folder on demand
  • No local manifest — can't answer "what was uploaded?"

Anchor

  • Resumes from the last confirmed 8 MB chunk, always
  • Full per-file progress in anchor status
  • Lightweight CLI — runs when you want, stops when you stop
  • Fully scriptable — pipe, cron, Task Scheduler
  • Watch any folder on demand with one command
  • SQLite manifest tracks every file: hash, size, status

Beta Access

Anchor is currently in closed beta. The Google OAuth app is in testing mode, which means only approved users can authenticate.

Submit your Gmail address and you'll be added as a test user manually. Once approved, you can download the JAR, place the provided credentials.json in ~/.anchor/, and start uploading.

No Google Cloud configuration needed on your end. Just download, place the file, and run.

  • 01 Submit your Gmail address using the form
  • 02 Wait for approval email (usually within 24h)
  • 03 Download the JAR and hasher.exe from the Download section
  • 04 Click Authenticate with Google — a StoredCredential file downloads automatically
  • 05 Move it to C:\Users\YourName\.anchor\tokens\StoredCredential
  • 06 Run java -jar anchor.jar add-folder "C:\Your\Folder" — no browser login needed!

> Quick Start

# 1. Install Java 17+ if not already installed
# 2. Create the Anchor config directories
mkdir $env:USERPROFILE\.anchor\tokens
 
# 3. Click "Authenticate with Google" above — a StoredCredential file downloads
Move-Item StoredCredential $env:USERPROFILE\.anchor\tokens\StoredCredential
 
# 4. Place hasher.exe in the config directory
Move-Item hasher.exe $env:USERPROFILE\.anchor\hasher.exe
 
# 5. Run Anchor — no browser login needed, tokens already saved
java -jar anchor.jar add-folder "C:\Your\Folder"
 
# 6. Check upload status anytime
java -jar anchor.jar status
